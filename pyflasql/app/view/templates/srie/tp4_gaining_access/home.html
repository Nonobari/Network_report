SRIE > TP4 - Gaining Access

The gaining access phase, also known as intrusion, is one of the key stages in a penetration test. It involves exploiting identified vulnerabilities in the target system to obtain unauthorized access. This step requires a deep understanding of the technologies, protocols, and tools used by the target. Penetration testers employ various techniques such as software exploits, input parameter manipulation, identity impersonation, social engineering, and other context-specific methods. The main objective of this phase is to achieve a successful compromise of the system, gaining elevated privileges or establishing a persistent presence for further activities. It is essential to document each step and adhere to ethical boundaries to ensure the integrity of the penetration testing process.

Once access has been gained, the next steps in the penetration testing process involve further exploration and exploitation of the compromised system. This may include escalating privileges, pivoting to other systems within the network, and conducting lateral movement to gather more information and access additional resources. The goal is to maximize the impact of the intrusion by expanding the attacker's control and potential damage. During this phase, the pentester may employ advanced techniques like post-exploitation frameworks, command-and-control infrastructure setup, and persistence mechanisms to maintain access even after detection or system reboot. Additionally, it is crucial to carefully document all actions taken during this phase, as it provides valuable insights into the vulnerabilities, weaknesses, and potential countermeasures that need to be addressed. Effective communication with the client throughout this process ensures transparency, facilitates understanding of the security risks, and supports the development of effective remediation strategies.

Useful tools used in this phase

Below is a list of tools that can be used for this pentesting phase:

  1. hydra

    Hydra is a very fast network logon cracker which supports many different services. Hydra is a parallelized login cracker which supports numerous protocols to attack. New modules are easy to add, beside that, it is flexible and very fast. This tool gives researchers and security consultants the possibility to show how easy it would be to gain unauthorized access from remote to a system.

  2. sqlmap

    sqlmap is an open source penetration testing tool that automates the process of detecting and exploiting SQL injection flaws and taking over of database servers. It comes with a powerful detection engine, many niche features for the ultimate penetration tester and a broad range of switches lasting from database fingerprinting, over data fetching from the database, to accessing the underlying file system and executing commands on the operating system via out-of-band connections.
